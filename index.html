DATA STRUCTURES

DATA SCTRUCTURE BEST FOR ADVANTAGES DISADVANTAGES
Array/List Storing ordered data with index access
  1. Fast access
  2. Simple to use
  3. Memory efficient
  1. Slow insert/delete (shifting needed)
  2. Fixed size (arrays)
Stack LIFO operations (Last In, First Out)
  1. Fast push/pop (O(1))
  2. Simple logic
  3. Good for recursion
  1. Limited access (only top element)
  2. Not suitable for searching
Queue FIFO processing (First In, First Out)
  1. Fair processing order
  2. Fast enqueue/dequeue (O(1))
  1. No random access
  2. Limited flexibility
Linked List Frequent insertions and deletions
  1. Easy insert/delete
  2. Dynamic size (O(1))
  3. No memory shifting
  1. Slow access (O(n))
  2. Extra memory for pointer
Graph Representing relationships or networks
  1. Models real-world connections
  2. Very flexible structure
  1. Can be complex
  2. Traversal can be expensive
